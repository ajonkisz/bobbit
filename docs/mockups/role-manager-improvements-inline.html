Role Manager — UX Improvement Proposals

Six targeted improvements to the list view. Each section shows current vs proposed side-by-side. Hover over rows to see interactive states.

1. Tighter Row Spacing

Reduce gap 6px→2px, padding 12px 18px→10px 16px, border-radius 10px→8px
Problem: Current rows feel sparse with 6+ roles. Change: Tighter gap and padding for a denser, scannable list without scrolling.
Current — gap: 6px, padding: 12px 18px
Proposed — gap: 2px, padding: 10px 16px

Drop the bordered square; show role slug as secondary info

Design Rationale

  • Density: 2px gaps + 10px padding — six roles fit without scrolling.
  • Visual hierarchy: Removing avatar box reduces noise; sprite is already a strong anchor.
  • Info density: Monospace slug gives power users the machine name at a glance.
  • Safety: Delete hidden at rest follows least-surprise principle for destructive actions.
  • Branding: Empty state uses Bobbit sprite instead of generic icon.
  • Mobile: 28px touch targets, both buttons visible (no hover), slug hidden.
  • Constraints: All colours use existing HSL variables. No new design tokens.
