Role Manager — UI Mockup

Roles are identified by pixel-art accessories (crown, bandana, magnifying glass, etc.), not by color. Color is session identity (Aurora Borealis palette), accessories are role identity.

1. Role Identity = Accessories
Each role gets a unique pixel-art accessory overlay rendered on top of the Bobbit sprite. Accessories are counter-hue-rotated so they keep their fixed colors regardless of the session's identity hue. Currently: crown (team-lead), bandana (coder). New accessories needed for reviewer, tester, and custom roles.

2. Role Manager Dialog — List View
Launched from the sidebar "Roles" button. Each role row shows the Bobbit with its accessory at small scale, the role name, built-in badge, and tool restriction summary. No color picker — accessories are the visual identity.

4. Accessory Gallery — Pixel Art at Multiple Hues

Key principle: The crown stays gold, the bandana stays red, the magnifying glass stays blue — regardless of whether the Bobbit body is green, purple, or teal. This is achieved by counter-rotating the accessory layer: if the body has hue-rotate(150deg), the accessory gets hue-rotate(-150deg) to cancel it out. Already implemented for crown and bandana in session-colors.ts.
5. Design Rationale
Accessory = role identity. Color = session identity. These are orthogonal axes. A coder always has a bandana whether the session is green, blue, or purple. This separation is already established by the crown/bandana system — the role manager just extends it with more accessories and makes it user-configurable.
Accessory overlay technique. Each accessory is a CSS box-shadow pixel grid rendered as a separate <span> positioned over the base Bobbit. Counter-hue-rotated to stay fixed. The role definition stores an accessory ID (e.g. "crown", "magnifier"), and statusBobbit() renders the matching overlay.
Accessory gallery in the selector. The edit dialog shows each accessory option as an emoji placeholder (fast to render, universally supported). The actual rendered pixel-art is visible in the sidebar and session list. Alternatively, the selector could render mini pixel-art previews — but emoji is simpler for a picker UI.
Custom roles can reuse or extend accessories. The built-in set (crown, bandana, magnifier, goggles, headphones, pencil, book, glasses, shield) covers common archetypes. Custom roles pick from this gallery. Adding new pixel-art accessories is a code change (adding a box-shadow definition), not a UI configuration.
No color picker in role editor. Color is managed per-session in the rename dialog (existing Aurora Borealis palette picker). The role editor only manages: name, label, prompt, tools, and accessory. This keeps the two identity axes cleanly separated.
Same dialog patterns as before. Imperative render-in-container, mini-lit Dialog/Button/Input, list-to-detail navigation with back arrow. Tool chips are interactive toggles. Built-in roles have read-only name + "Reset to Default" instead of delete.
