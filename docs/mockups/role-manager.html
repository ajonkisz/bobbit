Role Manager — UI Mockup

Design for the role management dialog. Shows list view, detail/edit view, tool selector, color picker, sidebar integration, and role assistant flow.

1. Role Manager Dialog — List View
Launched from the sidebar (new "Roles" button) or from Settings. Shows all roles — built-in and custom — with visual identity badges. Follows the imperative render-in-container pattern used by all Bobbit dialogs (confirmAction, showGoalDialog, etc.).

2. Role Detail / Edit View
Clicking a role in the list navigates to this detail view. The dialog transitions (no separate page — same dialog, different content). "Back" returns to the list. Built-in roles show a "Reset to Default" button instead of Delete.

Tool selector: Discovered dynamically via GET /api/tools. Each chip toggles on click. Active chips get green border + text. The reviewer role demonstrates restricting Write/Edit/Delegate tools — reviewers can read and search but not modify code.

Color picker: Uses the same 20-entry Aurora Borealis palette from session-colors.ts (hue-rotated Bobbit). Each swatch shows the Bobbit at that hue. The selected color is used for the role's avatar in the list and sidebar badges.

Built-in roles: Name field is read-only. Delete is replaced with "Reset to Default" which restores the original prompt and tool set from swarm-prompts.ts.

5. Design Rationale
Follows existing patterns exactly. The dialog uses the same imperative render-in-container pattern as showGoalEditDialog, showRenameDialog, and showQrCodeDialog. No new UI primitives needed — just mini-lit Dialog, Button, Input.
Color is identity, not state. The hue-rotate palette from session-colors.ts is reused for role identity colors. Status colors (yellow=starting, red=terminated, green=canonical) remain unchanged and orthogonal to role color.
Tool chips are toggleable, not a multi-select. Each tool is an independent toggle — more scannable than a scrollable checkbox list. Green border = active. Empty = all tools (explicit in the UI hint text).
List → Detail navigation in-dialog. Clicking a role replaces the list with the edit form (with a back arrow). No nested dialogs or modals. This matches how SessionListDialog works — one dialog, switching views.
Role assistant mirrors goal assistant. "Create with Assistant" launches a special session with ROLE_ASSISTANT_PROMPT, outputs <role_proposal> blocks. Same parse + callback pattern as onGoalProposal.
Built-in roles are protected. Name is read-only, delete is disabled. "Reset to Default" restores the original prompt from swarm-prompts.ts. This prevents accidental breakage of core swarm functionality.
