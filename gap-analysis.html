Bugs Found

BUG 1: Goal field wipe on partial update — GoalStore.update() was using Object.assign with undefined values, wiping fields not included in PUT body. FIXED in commit b88f057. However, the current goal a0f4cc4e still has corrupted data (missing spec, cwd, swarm fields).
BUG 2: repoPath corruption — The direct JSON edit of gateway-goals.json produced C:Usersjsubrwobbit (mangled backslashes). The API update fixed it to the correct value but the original corruption persists in the repoPath field.
BUG 3: execSync ENOENT on Windows — createWorktree() in git.ts uses execSync() which looks for cmd.exe. In Git Bash/MSYS2 environments, C:\Windows\system32\cmd.exe may not be on PATH. Needs shell: true or explicit shell path.
BUG 4: Team Lead env vars — Role prompts reference BOBBIT_GATEWAY_URL, BOBBIT_AUTH_TOKEN, BOBBIT_GOAL_ID for REST API calls. These must be passed as environment variables when spawning the agent subprocess. Needs verification that SwarmManager.startSwarm() actually sets them.

Missing / Not Implemented

  • Merge conflict resolution — spec says Team Lead handles them, but no guidance or tooling exists
  • Goal completion flow — /swarm/complete endpoint exists but end-to-end cleanup (terminate all agents, clean all worktrees, mark goal done) is unverified
  • Max concurrent agents enforcement — maxConcurrent: 5 is stored but unclear if spawnRole() checks it before spawning
