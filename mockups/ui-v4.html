Vision Active

Mission Statement All agentscontext/vision.yaml

Build the simplest, most powerful analytics platform for small B2B teams. We believe data-driven decisions shouldn't require a data team.

Product Vision Summary All agentscontext/vision.yaml

A single dashboard where small teams feel at home. Self-serve onboarding, team workspaces with role-based access, usage-based billing, and an API developers actually enjoy. A new user should see their first insight within 5 minutes.

What We Are All agentscontext/vision.yaml

The fastest path from "I need data" to "I see the answer." A lightweight, opinionated analytics tool for small teams who don't have (and don't want) a data team.

What We Are Not All agentscontext/vision.yaml
  • Not an enterprise BI tool — No 50-page reports, no OLAP cubes
  • Not a data warehouse — We connect to your data, we don't replace where it lives
  • Not a notebook — No SQL editor, no Python runtime

Design product/design.md

Design Principles
  • Speed over features — A fast app with fewer features beats a slow one with many.
  • Convention over configuration — Smart defaults. Expose settings only when needed.
  • Show, don't tell — Live previews, inline docs. No "learn more" links.
Design Guardrails
Every screen loads in < 200ms

No external CSS frameworks — design system tokens only

Architecture product/architecture.md

Technical Guardrails
No passport.js — Direct OAuth flows for full control
No ORMs — Raw SQL with parameterized queries only
Test coverage ≥ 80% — All PRs must meet threshold
Technical Patterns
  • API envelope — { data, error, meta } on all responses
  • Middleware — Composable: requireAuth, optionalAuth, requireRole
  • Components — Max 3 nesting levels. No prop drilling past 2.
Sitemap
/ (App Shell)
/dashboard — KPI cards, charts
/explore — Data explorer (no SQL)
/team — Members, roles, invites
/settings — Account, billing, API keys
/ (Public)
/login · /register · /shared/:token
Navigation Rules
  • Sidebar always visible. Max 6 top-level items.
  • No nested navigation — sub-pages use tabs within page body.
  • Command palette (Cmd+K) available everywhere.

Glossary product/glossary.yaml

Shared vocabulary. Agents must use these exact terms in code, UI copy, and docs.

Term
Definition & Usage
Workspace
Top-level container for a team's data. Never "organization" or "tenant" in UI.
Dashboard
A saved view of charts and KPI cards. Not "report" or "view".
Insight
A single visualization or metric card. Not "widget" or "tile".
Member
A person in a workspace. Not "user" (internal only) or "collaborator".
Event
A data point sent to the ingestion API. Not "log" or "record".
Alert
A threshold-based notification. Not "alarm" or "trigger".
